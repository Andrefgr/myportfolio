Deliberative behavior based on demand in state spaces

  Agent design and implementation to collect targets, based on automatic reasoning based on state space search, capable of minimize the distance traveled when collecting targets.
  Deliberative agents are more complex than reactive agents since are not limited to reacting to stimulus from the environment, these agents are capable to taking decisions based on logical reasoning.
  A notable failure is the fact that the agent passes targets that exist at a shorter distance than the target it intends to hit. This is due to the fact that the targets in the world are randomly distributed, and the agent addresses each one in the order in which they are distributed. However, if he passes over a target that is not the intended one, he captures it and continues his process without any errors.

Deliberative behavior based on Markov's decision processes

  Agent design and implementation to collect targets, based on Markov's decision processes, capable of minimizing the distance traveled at collection of the targets.
  They are called Markov processes because the modeled processes obey Markov's property: the effect of an action in a state depends only on the action and the current state of the system and not on the path that led the process to that state. That means, the prediction of the following states only depends on the present state.
  Here as he knows the location of each objective, the only factor in his trajectory is his current position. This allows an optimal solution to be obtained. However, whenever the agent acquires a target, it has to re-plan its trajectory, which can cause a drastic increase in temporal complexity when there is a high number of targets.

Learning by Reinforcement

  Agent design and implementation to collect targets, capable to learn from experience, based on reinforcing learning mechanisms.
  In this algorithm the agent through his perception interacts with the environment, observing the usefulness actions that he develops through motivation and associating this reinforcement to his memory of learning. The more experience you have, the more strengthened this memory will be and the more capacity the agent will have to act beneficially.
  As you can see the agent begins by moving randomly around the world, colliding with obstacles, but receiving the reinforcement and storing it in memory. After some time, the memory ends up building up, with the movements that direct the agent to the target indicated in green.

Architecture (cont.)

  The Arduino Mega is the main module of the system, being the only one to know all the secondary modules, the Arduinos Uno.
  Between NodeMCU and the lower layer, to which the Arduino Mega belongs, communication is supported by cable (TX / RX). Then, the Arduino Mega and the Arduinos Uno, transmit data via bluetooth.
  Finally, sensors are connected to each Arduino Uno, all by connecting cables to digital or analog pins, depending on the physical quantities they measure.
